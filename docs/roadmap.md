# Roadmap

## Current Focus

MahaGuardian Community Edition is focused on making the trust boundary, policy
enforcement, and audit model concrete and testable.

Current visible priorities:

- stable enforcement pipeline
- public-facing security documentation
- clean CI and open-source project signals
- externally legible identity, authorization, and audit controls

## Near-Term

### 1. Public Evidence Surface

- strengthen README security and governance signals
- simplify docs structure for external scanners and evaluators
- improve website security and architecture visibility

### 2. Runtime and Demo Packaging

- strengthen documented local/demo run path
- improve deployment guidance
- make more of the enforcement flow visible in examples

### 3. Observability

- improve public documentation of auditability and operator review flows
- expose more observability signals in docs and examples

## Planned

- stronger multi-Guardian / federation story
- richer operator workflows around approval and escalation
- tighter deployment packaging for production-style demos
- clearer enterprise packaging and commercial boundary

## Non-Goals For The Current Public Repo

- full hosted product packaging
- full enterprise control plane
- publishing internal review packets or patent strategy material

## Success Criteria

The public repo should read as:

- a runnable project
- a security-conscious open-source system
- a clearly documented agent-infrastructure architecture
- a credible foundation for further enterprise development
